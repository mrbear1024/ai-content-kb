# 在知识库中进行 AI 内容创作

本指南说明如何使用 `ai-content-kb` 创作文章、课程、Newsletter、社交媒体内容和视频脚本。

[English](CONTENT_CREATION_GUIDE.md)

## 核心原则

先检索和核对原文，再组织观点和写作；先生成 staging 草稿，再由人决定是否进入正式创作与发布流程。

```text
明确创作任务
  -> 检索 wiki/ 和 .kb/links/
  -> 回到 raw/、sources/、products/ 阅读原文
  -> 区分个人判断、外部主张、已有表达和新推断
  -> 生成资料清单与大纲
  -> 写入 .kb/staging/drafts/
  -> 事实、引用与风格审核
  -> raw/drafts/ 继续发展，或进入 products/ 发布
  -> 将新概念和关系 backfill 回知识库
```

## 1. 给出清楚的创作任务

尽量提供：

- 主题和核心问题；
- 目标读者；
- 内容形式与预计长度；
- 希望达到的目标；
- 语气、风格与必须保留的个人观点；
- 可以使用或不能使用的材料范围。

```text
在知识库中进行 AI 内容创作：
写一篇面向 AI 产品经理的中文文章，主题是 Context Engineering。
长度约 2500 字，重点说明它与 Prompt Engineering 的区别。
优先使用我自己的判断和已经审核的外部来源；所有关键观点注明仓库路径。
先生成资料清单和大纲，不要直接写全文。
```

## 2. 先做资料与观点盘点

Codex 应先查 `wiki/` 和 `.kb/links/` 定位材料，再回到原始文件确认上下文。

| 内容类型 | 典型位置 | 创作中的作用 |
|---|---|---|
| 个人原创判断 | `raw/` | 文章立场、独特观点和个人经验 |
| 外部证据 | `sources/` | 事实、研究、定义和第三方案例 |
| 已发布表达 | `products/` | 可复用框架、案例和已有表述 |
| 知识索引 | `wiki/` | 快速定位概念和相关材料 |

```text
根据刚才的资料，输出“可直接使用的观点、需要补证据的观点、
可能重复的旧内容、值得新增的观点”四个列表。
```

## 3. 先生成大纲，再写草稿

```text
根据已确认的资料生成文章大纲。每一节列出核心结论、
使用的案例和来源路径；不要添加知识库中没有依据的事实。
```

确认大纲后：

```text
按照这个大纲写初稿，保存到 .kb/staging/drafts/。
保留我的表达立场，把外部作者的主张明确标注为外部观点。
文末增加 Sources 和“待人工确认”两个部分。
```

## 4. 审核 AI 草稿

至少完成四类检查：

1. **事实检查**：数字、定义和引述能否回到原文；
2. **归属检查**：个人判断、外部观点和 AI 推断是否清楚区分；
3. **重复检查**：是否与已有文章或课程重复；
4. **表达检查**：结构、语气和案例是否符合发布场景。

```text
审查这个 staging 草稿：逐项列出缺少来源、过度推断、
与旧内容重复、表达不清和需要我做决定的地方。
先报告，不要直接改稿。
```

## 5. 进入创作或发布流程

- 继续发展的内容：移动到 `raw/drafts/`；
- 已经人工确认、准备发布的内容：进入 `products/`；
- 仍有证据或表达问题：保留在 `.kb/staging/drafts/`。

发布后：

```text
回填知识库：分析这篇已发布文章中新出现的概念、案例和关系，
生成 Wiki 与 sidecar 候选，但不要修改已发布正文。
```

## 其他内容形式

### 课程大纲

```text
在知识库中进行 AI 内容创作：基于现有 Agent Engineering 材料设计一门 6 讲课程。
先分析已有课程和文章的重复内容，再输出学习目标、章节依赖、案例和知识缺口。
写入 .kb/staging/course-drafts/。
```

### 视频脚本

```text
在知识库中进行 AI 内容创作：把我关于 Harness Engineering 的个人判断
整理成一个 8 分钟视频脚本。区分我的观点与外部引用，给出画面提示，
并把初稿写入 staging。
```

### 社交媒体内容

```text
在知识库中进行 AI 内容创作：从这篇已审核文章提炼 5 条独立短内容。
不要改变原观点，不要补充未经来源支持的数据；
标明每条内容对应的原文路径。
```
